# 音乐开发文档

## 本次修改了什么

这次主要是修复个人网站里 B 站音乐不能播放的问题，并把原来不稳定的临时 `.m4s` 链接改成通过 BV 号播放。

修改内容：

1. 新增 B 站音乐代理接口

   文件：`src/app/api/bilibili-music/route.ts`

   作用：

   - 浏览器不再直接请求 B 站 `.m4s` 音频地址。
   - 页面请求本网站自己的接口 `/api/bilibili-music`。
   - 后端接口再去解析 BV 号，获取 B 站音频地址。
   - 后端把音频流转发给浏览器播放。
   - 支持 `Range` 请求，所以浏览器可以正常加载、缓冲和播放音频。
   - 响应头固定为 `audio/mp4`，避免浏览器因为 B 站返回 `video/mp4` 或 `application/octet-stream` 而拒绝播放。

2. 新增音乐播放器组件

   文件：`src/sections/plan/components/MusicPlayer.tsx`

   作用：

   - 显示右下角 BGM 播放按钮。
   - 支持传入 `bvid`。
   - 点击播放时自动请求 `/api/bilibili-music?action=stream&bvid=...`。
   - 不再直接把 B 站 `.m4s` 地址塞进 `<audio>` 标签。
   - 播放失败时会在按钮旁边显示错误提示。

3. 修改 2026 年 5-7 月计划页面

   文件：`src/sections/plan/MayJulyPlanSection.tsx`

   作用：

   - 页面底部音乐从临时 `.m4s` 链接改为 BV 号：

     ```tsx
     <MusicPlayer bvid="BV1AD421K7Ha" />
     ```

   - 音乐按钮位置改到右下角。
   - “返回计划”按钮移动到顶部图片左上角，不再显示在内容区下面。

## 为什么不能直接用 B 站 `.m4s` 链接

B 站搜索出来的音频地址通常是 `.m4s` 临时签名链接。

这种链接有几个问题：

- 有过期时间，过期后就不能播放。
- 有防盗链限制，浏览器直接播放容易失败。
- 返回的 `Content-Type` 不一定适合 `<audio>` 标签。
- 链接很长，不适合长期写进代码。

所以更稳定的方式是保存 BV 号，每次播放时再实时解析音频地址。

## 以后怎么换音乐

只需要改这个文件：

`src/sections/plan/MayJulyPlanSection.tsx`

找到：

```tsx
<MusicPlayer bvid="BV1AD421K7Ha" />
```

把里面的 BV 号换成新的即可：

```tsx
<MusicPlayer bvid="新的BV号" />
```

例如：

```tsx
<MusicPlayer bvid="BVxxxxxxxxxx" />
```

## 音乐接口说明

### 解析 BV 号

请求：

```text
/api/bilibili-music?action=media&bvid=BV1AD421K7Ha
```

作用：

- 根据 BV 号解析出当前可用的音频地址。
- 返回给前端一个同源代理地址。

### 播放音频流

请求：

```text
/api/bilibili-music?action=stream&bvid=BV1AD421K7Ha
```

作用：

- 实时解析 BV 号。
- 请求 B 站音频地址。
- 把音频流转发给浏览器。

## 本地验证方式

启动项目：

```bash
npm run dev -- -H 127.0.0.1 -p 3000
```

打开页面：

```text
http://127.0.0.1:3000/zh/plan/2026/may-july
```

验证音频接口：

```bash
curl -I -r 0-1023 "http://127.0.0.1:3000/api/bilibili-music?action=stream&bvid=BV1AD421K7Ha"
```

正常结果应该包含：

```text
HTTP/1.1 206 Partial Content
content-type: audio/mp4
```

## 常见问题

### 点播放没有声音

先确认 BV 号是否正确。

可以打开：

```text
http://127.0.0.1:3000/api/bilibili-music?action=media&bvid=BV1AD421K7Ha
```

如果返回 `200`，说明 BV 号能解析。

### 构建时报 `.next` 相关错误

如果 dev server 正在运行时又执行 `npm run build`，可能会出现 `.next` 被同时写入的问题。

处理方式：

1. 先停止当前 `npm run dev`。
2. 再执行：

   ```bash
   npm run build
   ```

### 为什么推荐 BV 号

BV 号是稳定的视频 ID。

临时 `.m4s` 链接会过期，但 BV 号不会因为签名过期而失效。页面保存 BV 号后，每次播放都会重新获取新的音频地址。
