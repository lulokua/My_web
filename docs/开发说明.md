# 开发说明

## 项目简介

本项目使用以下技术栈进行开发：

- **Next.js**（React 全栈框架，使用 App Router）
- **TypeScript**（类型安全的 JavaScript 超集）
- **Tailwind CSS**（原子化 CSS 框架）

## 环境要求

- Node.js 18.18 或更高版本
- npm / pnpm / yarn 任选其一（以下示例使用 npm）

## 安装依赖

在项目根目录执行：

```bash
npm install
```

## 常用命令

| 命令 | 说明 |
| --- | --- |
| `npm run dev` | 启动本地开发服务器（默认 http://localhost:3000） |
| `npm run build` | 构建生产环境产物 |
| `npm run start` | 运行生产环境构建结果 |
| `npm run lint` | 运行 ESLint 检查（如已配置） |

## 字体配置说明

本项目全局使用了自托管的开源字体 **MiSans**（小米字体）。

> **关键概念：** MiSans 是一款**自带中英文混合支持**的全球化字体，因此不需要将中文和英文拆分成不同的字体文件来使用。你提供的 4 个 `.woff2` 文件实际上代表了该字体的 **四种粗细（Weight）**，而不是分别用于中文和英文。

目前已在 `src/app/globals.css` 中引入了以下字重配置，并在 `tailwind.config.ts` 中设为默认的 `sans` 字体：

1. **Light (300)**: `MiSans-Light.woff2` —— 用于细体文字。
2. **Regular (400)**: `MiSans-Regular.woff2` —— 用于标准正文（中英通用）。
3. **Demibold (600)**: `MiSans-Demibold.woff2` —— 用于半粗体/小标题。
4. **Bold (700)**: `MiSans-Bold.woff2` —— 用于粗体/大标题。

**如何使用？**
开发时不需要特别关注中英文的切换，直接在 Tailwind CSS 中使用标准的字重工具类即可自动匹配对应字体文件：
- `font-light` -> 会自动加载 Light (300)
- `font-normal` -> 会自动加载 Regular (400)
- `font-semibold` -> 会自动加载 Demibold (600)
- `font-bold` -> 会自动加载 Bold (700)

### 例外字体：Anthropic（仅首页 Hero）

首页 Hero 大标题使用了一款**纯英文**的视觉字体 `Anthropic`（`Anthropic-English.ttf`），仅作用于 `src/sections/home/HomeHero.tsx` 的 `<h1>`，通过 Tailwind 自定义类 `font-anthropic` 调用。

约束：

- 该字体**只含英文字形**，所以中文站点访问首页时，标题里的中文字符会通过字体回退渲染。`tailwind.config.ts` 中已为 `anthropic` 字体栈追加了 `MiSans` 作为 fallback，确保中文环境下的标题仍以 MiSans 呈现。
- 除首页 Hero 外，**任何其他页面与组件都不应使用 `font-anthropic`**。其余位置一律使用默认字体（即 MiSans）。

## 目录结构

```
My_web/
├── src/
│   └── app/
│       ├── layout.tsx     # 根布局
│       ├── page.tsx       # 首页（当前为 Hello World）
│       └── globals.css    # 全局样式（包含 Tailwind 指令）
├── docs/
│   └── 开发说明.md        # 本文档
├── public/                # 静态资源（按需创建）
├── next.config.ts         # Next.js 配置
├── tailwind.config.ts     # Tailwind CSS 配置
├── postcss.config.mjs     # PostCSS 配置
├── tsconfig.json          # TypeScript 配置
└── package.json
```

## 路径别名

`tsconfig.json` 中已配置 `@/*` 指向 `src/*`，例如：

```ts
import Foo from "@/components/Foo";
```

## 开发流程

1. 在 `src/app/` 下添加页面或路由（App Router 约定式路由）。
2. 公共组件建议放在 `src/components/`。
3. 使用 Tailwind 工具类编写样式，必要时在 `globals.css` 中扩展。
